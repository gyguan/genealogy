# 12. MVP1 待实现 Todo List

本文用于跟踪 Genealogy MVP1 从“可运行验证版”走向“可验收交付版”仍建议补齐的事项。当前后端主流程、本地 API 验证、数据库迁移、演示数据、静态前端 MVP 已完成；本清单聚焦后续需要继续完善、验收和工程化的内容。

## 一、使用说明

- 状态标识：
  - `[ ]` 未开始
  - `[~]` 进行中
  - `[x]` 已完成
- 优先级：
  - `P0`：MVP1 验收前必须完成或确认
  - `P1`：建议 MVP1 验收前完成，影响体验和质量
  - `P2`：MVP1 后增强，不阻塞验收
- 建议推进顺序：先验收脚本与部署联调，再补权限/审核/前端工程化，最后做复杂规则与运维增强。

## 二、P0：MVP1 验收前必须完成或确认

### 1. 端到端验收脚本

- [ ] 编写一份完整 API 验收脚本，覆盖注册、登录、创建宗族、创建支派、创建字辈、录入人物、建立关系、绑定来源、上传附件、提交审核、审核通过、查看世系、导入导出、查询日志。
- [ ] 将验收脚本放到仓库，例如：`docs/test/mvp1-api-acceptance.md` 或 `backend/genealogy-backend/scripts/mvp1-api-test.sh`。
- [ ] 明确每一步输入参数、预期响应、失败排查方式。
- [ ] 输出一份“本地验收通过记录”，记录测试环境、数据库版本、后端 commit、测试人、测试时间。

验收标准：

```text
从空数据库启动后，可以按脚本完整跑通 MVP1 主流程，无需手动修改数据库。
```

### 2. 本地部署说明最终确认

- [ ] 更新并确认 `backend/genealogy-backend/README.md` 中的本地启动步骤。
- [ ] 确认 `docker-compose.yml`、`application.yml`、README 中数据库连接信息完全一致。
- [ ] 补充“旧数据库清理方式”：`docker compose down -v`。
- [ ] 补充“Flyway 报 schema 非空但无 history 表”的处理方式。
- [ ] 补充“导入演示数据”的命令和演示账号。

验收标准：

```text
新开发者只看 README，即可在本地完成数据库启动、后端启动、Swagger 打开和演示账号登录。
```

### 3. 数据库迁移脚本最终梳理

- [ ] 检查 `db/migration` 下是否仍存在重复版本号、无效脚本或历史冲突脚本。
- [ ] 整理 `V1` 到最新版本的迁移清单，标注每个脚本职责。
- [ ] 确认全量空库迁移通过。
- [ ] 确认旧本地库升级迁移通过，尤其是 `operation_log.detail` 类型修复脚本。
- [ ] 明确 `db/seed/demo-data.sql` 为手动演示数据，不自动执行。

验收标准：

```text
空库启动自动建表成功；已有本地库重启迁移成功；JPA validate 通过。
```

### 4. API 权限边界复测

- [ ] 匿名访问写接口应返回未登录错误。
- [ ] 非宗族成员访问宗族写接口应返回无权限错误。
- [ ] 普通 editor 不能审核通过/驳回。
- [ ] clan_admin 可以审核通过/驳回。
- [ ] 未登录或非成员查看在世人员，应触发敏感字段脱敏。
- [ ] 宗族成员查看在世人员，可以看到完整字段。

验收标准：

```text
权限、角色、隐私脱敏均符合 MVP1 规则，无越权写入和敏感字段泄露。
```

### 5. 附件上传下载本地验证

- [ ] 验证 `POST /api/v1/clans/{clanId}/attachments/upload` 可以上传图片、PDF、txt。
- [ ] 验证超过 20MB 文件会被拒绝。
- [ ] 验证不支持的文件类型会被拒绝。
- [ ] 验证 `GET /api/v1/attachments/{attachmentId}/download` 可以下载。
- [ ] 验证非宗族成员不能下载附件。
- [ ] 验证 `./data/attachments` 路径下文件真实存在。

验收标准：

```text
附件元数据、物理文件、权限校验、下载响应均正常。
```

## 三、P1：建议 MVP1 验收前完成

### 1. 正式前端工程化准备

- [ ] 评估是否继续使用 `frontend/mvp` 静态页面作为验收入口，还是升级为 React/Vue 工程。
- [ ] 若升级前端工程，创建正式目录，例如 `frontend/genealogy-web`。
- [ ] 补充路由、API Client、登录态存储、全局错误提示、页面状态管理。
- [ ] 将当前静态 MVP 页面能力迁移为正式页面。
- [ ] 支持 `.env` 配置后端 API 地址。

验收标准：

```text
前端可以完成 MVP1 主流程演示，不依赖手动拼 curl 或 Swagger。
```

### 2. 审核 diff 结构化

- [ ] 当前审核 `diffSummary` 为文本，建议新增结构化 before/after payload。
- [ ] 支持人物、关系、来源、支派、字辈方案的变更前后对比。
- [ ] 审核任务详情接口返回 diff 数据。
- [ ] 前端审核页面展示字段级变更。

验收标准：

```text
审核人能看清楚“改了什么”，而不是只看到一段说明文本。
```

### 3. branch_admin 支派范围权限

- [ ] 当前已具备角色基础，但支派范围权限仍需细化。
- [ ] branch_admin 只能管理授权支派及其下级支派。
- [ ] editor/viewer 可根据 scope_type/scope_id 控制访问范围。
- [ ] 人物、关系、来源、世系、导入导出均接入支派范围判断。
- [ ] 补充越权测试用例。

验收标准：

```text
不同角色只能访问和操作授权范围内的数据。
```

### 4. 关系规则继续增强

- [ ] 细化继嗣关系规则。
- [ ] 细化出嗣关系规则。
- [ ] 细化过继/养子关系规则。
- [ ] 增加循环关系检测，避免祖先成为自己的后代。
- [ ] 增加配偶关系删除时反向关系同步处理。
- [ ] 增加关系冲突检查接口，导入前可批量预检。

验收标准：

```text
族谱核心关系不产生明显语义冲突，导入和手工录入规则一致。
```

### 5. 导入导出增强

- [ ] 当前支持 Excel 可打开的 CSV，后续可评估是否引入 `.xlsx`。
- [ ] 增加导入错误报告下载。

- [ ] 增加大文件导入进度查询。
- [ ] 增加导入批次记录表，便于追溯。

验收标准：

```text

```

### 6. 世系图体验增强

- [ ] 当前前端为简单节点/边展示，后续增加正式树图布局。

- [ ] 支持按支派筛选。

验收标准：

```text

```

```text

2. 本地 API 验收脚本完整跑通。
3. 演示数据可手动导入，演示账号可登录。
4. 宗族、支派、字辈、人物、关系、来源、附件、审核、世系、导入导出、日志主流程可用。

6. README 和数据库文档足以支持新开发者独立启动。
7. Backend CI 持续通过。
8. 已明确 P1/P2 延后项，不阻塞 MVP1 交付。
```
