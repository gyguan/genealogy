# 01. MVP 1 需求设计

## MVP 目标

MVP 1 先完成族谱系统的数据底座和协作闭环：宗族、支派、字辈、人物、亲属关系、资料来源、审核、世系查看和基础导入导出。

## 范围内模块

| 模块 | 说明 |
|---|---|
| 宗族管理 | 创建宗族，维护姓氏、堂号、郡望、始祖、祖籍地等信息 |
| 支派管理 | 支持多级支派、始迁祖、迁徙地、支派负责人 |
| 字辈管理 | 支持派语、世次映射、宗族级和支派级字辈 |
| 人物档案 | 支持姓名、谱名、字号、世次、字辈、排行、生卒、传记、墓葬 |
| 关系管理 | 支持父母、配偶、子女、继嗣、出嗣、养父母等关系 |
| 资料来源 | 支持老谱、口述、墓碑、照片、地方志等来源绑定 |
| 审核中心 | 支持新增、修改、删除的审核和变更对比 |
| 世系图 | 支持家庭图、上溯图、下延世系树 |
| 导入导出 | 支持 Excel 导入、导入校验、人物和关系导出 |
| 权限隐私 | 支持角色权限、支派范围和在世人员字段隐藏 |

## 暂不纳入 MVP 1

- 老谱 OCR 自动识别
- AI 关系抽取
- 自动寻祖推荐
- 高级谱书排版
- 宗亲社区
- 捐赠和活动管理

## 核心流程

```text
创建宗族 → 创建支派 → 维护字辈 → 录入人物 → 建立关系 → 绑定来源 → 提交审核 → 正式入库 → 查看世系
```

## 验收标准

1. 可以创建宗族空间并维护基础信息。
2. 可以创建多级支派并指定负责人。
3. 可以维护字辈派语并进行录入提示。
4. 可以新增、编辑、查询人物档案。
5. 可以维护父母、配偶、子女、继嗣、出嗣关系。
6. 可以从任意人物生成基础世系图。
7. 可以上传资料来源并绑定到人物或关系。
8. 提交资料审核通过后才能进入正式谱库。
9. 可以按支派导入和导出人物数据。
10. 在世人员敏感信息可按权限隐藏。
