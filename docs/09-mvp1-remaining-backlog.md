# 09. MVP 1 剩余建设 Backlog

本文用于暂存 MVP 1 剩余建设内容，并作为后续逐步实现的任务基线。

## 一、当前基线

当前后端已经完成数据底座主干，并通过启动级验证：

- Maven 编译与测试
- PostgreSQL 拉起

## 二、P0：MVP 闭环必须完成

### P0-1 权限隐私 MVP 化

目标：让系统从“可匿名调用”升级为“多人协作时具备基础访问控制”。

建设内容：

1. 接口登录态强校验
   - 从可选 `Authorization` 逐步切换为核心写接口强制登录。
   - 未登录访问核心写接口时返回 `AUTH_UNAUTHORIZED`。

2. 宗族成员校验
   - 用户必须是宗族成员才能访问宗族内数据。
   - 非成员禁止创建、修改、审核、导入、绑定来源等操作。

3. 角色权限校验
   - clan_admin：宗族管理员，可维护宗族、支派、成员、审核。
   - branch_admin：支派管理员，可维护授权支派范围内人物和关系。
   - editor：采集/编辑人员，可提交资料但不能审核。
   - viewer：只读成员。

4. 支派范围控制
   - 用户只能维护授权支派范围内的人物、关系和资料。
   - 支派管理员不能越权修改其他支派。

5. 在世人员隐私字段隐藏
   - 对无权限用户隐藏在世人员敏感字段。
   - 优先隐藏：birthDate、birthPlace、residencePlace、biography、tombPlace、epitaph 等。

验收标准：

- 未登录不能执行核心写操作。
- 非宗族成员不能访问宗族内部数据。
- 普通成员不能审核资料。
- 支派管理员不能越权维护其他支派。
- 无权限用户查询在世人员时，敏感字段被隐藏。

### P0-2 审核覆盖扩展

目标：把当前“人物审核”扩展为 MVP 范围内的资料变更审核。

建设内容：

1. 关系审核
   - 关系新增、修改、删除支持提交审核。
   - 审核通过后关系进入正式状态。

2. 来源证据审核
   - 来源创建、来源绑定、解除绑定支持审核。
   - 关键证据链变更需要审核留痕。

3. 字辈变更审核
   - 字辈方案和字辈明细变更支持审核。

4. 支派变更审核
   - 支派新增、修改、删除支持审核。

5. 变更对比
   - `AuditRecord` 补充 before/after payload。
   - 审核页面可展示变更摘要和核心差异。

验收标准：

- 人物、关系、来源、字辈、支派关键变更均能提交审核。
- 审核通过后才进入正式数据状态。
- 审核驳回后回到草稿或保持原状态。
- 审核记录可追溯变更人、审核人、变更摘要和处理结果。

### P0-3 真实附件上传能力

目标：把当前“附件元数据登记”升级为可用的资料上传能力。

建设内容：

1. 文件上传接口
   - 支持 multipart 文件上传。
   - 上传后生成附件记录。

2. 文件存储
   - MVP 阶段可先支持本地存储。
   - SAE/云部署阶段支持 OSS 存储。

3. 文件访问
   - 支持附件下载。
   - 支持图片类附件预览 URL。

4. 安全限制
   - 文件大小限制。
   - 文件类型白名单。
   - checksum 计算。
   - 权限校验。

验收标准：

- 用户可以上传资料图片或文档。
- 附件可绑定到资料来源。
- 可下载或预览附件。
- 非授权用户不能访问附件。

### P0-4 前端最小闭环页面

目标：形成可以演示的 MVP 产品闭环。

最小页面：

1. 登录/注册
2. 宗族列表与宗族创建
3. 支派树维护
4. 字辈方案维护
5. 人物列表与人物编辑
6. 关系维护
7. 资料来源与来源绑定
8. 审核任务列表与审核处理
9. 世系图查看
10. CSV 导入导出
11. 操作日志查看

验收标准：

- 能通过页面完成 MVP 主流程。
- 页面能展示审核状态、世系结果和来源证据。
- 普通用户与管理员看到的操作入口不同。
