# Genealogy 查询类页面实现规范

> 本文是 `docs/21-frontend-page-pattern-spec.md` 的查询列表页专项规范，适用于人物档案、宗族文化、来源资料、审核记录、成员管理、操作日志等“条件筛选 + 结果列表”页面。
>
> 查询类页面实现、评审和验收时，以本文为直接依据；如与通用规范存在表达差异，查询类页面优先采用本文约定。

## 1. 目标与适用范围

查询类页面围绕以下核心任务设计：

1. 快速设置高频查询条件；
2. 按需展开低频筛选条件；
3. 清晰展示查询结果、状态和分页；
4. 在正确层级提供创建、导出、批量和行级操作；
5. 保留查询现场，支持刷新、返回、分享和异常恢复。

适用页面包括但不限于：

- 人物档案查询；
- 宗族文化资料查询；
- 来源资料查询；
- 修谱任务查询；
- 审核记录查询；
- 成员与权限查询；
- 操作日志查询。

不适用于画布工作台、分步向导、数据看板、复杂编辑页等非列表主任务页面。

---

## 2. 页面标准结构

查询类页面统一采用上下双 Card 结构，不增加无业务价值的顶部介绍 Card。

```text
查询 Card
├─ 业务查询标题
├─ 基础查询条件（默认一行）
├─ 更多筛选条件（按需展开）
└─ 更多筛选、重置、查询

结果 Card
├─ 查询结果标题
├─ 页面级业务操作
├─ 查询状态与结果说明
├─ Table / Mobile Card List
└─ 总数与分页
```

标准原则：

- 查询 Card 标题使用具体业务名称，例如“人物档案查询”，不使用模糊的“查询条件”；
- 结果 Card 标题统一使用“查询结果”；
- 页面级业务操作放在结果 Card 标题右侧；
- 查询条件与结果列表不得混入同一个 Card；
- 基础条件和更多筛选之间不使用横向分割线，通过间距表达层级。

## 3. 查询 Card 规范

### 3.1 标题

查询标题放在查询 Card 内部左上角。

推荐：

- 人物档案查询；
- 来源资料查询；
- 审核任务查询；
- 宗族文化查询。

禁止：

- 页面外单独再放同名标题；
- Card 标题使用“查询条件”“筛选”等模糊名称；
- 页面顶部重复展示介绍性大卡片。

### 3.2 基础查询条件

基础查询条件承载最高频、最关键的筛选项，桌面端默认只占一行。

人物档案标准字段：

| 顺序 | 字段 | 控件 | 说明 |
|---|---|---|---|
| 1 | 宗族 | 单选下拉框 | 页面核心业务范围 |
| 2 | 支派 | 单选下拉框 | 根据宗族联动加载 |
| 3 | 姓名 | 文本输入框 | 支持精确或模糊姓名 |
| 4 | 关键词 | 文本输入框 | 支持姓名、别名、谱名、字号等综合查询 |

布局要求：

- 字段按“业务范围 → 精确条件 → 综合条件”从左到右排列；
- 桌面端默认一行展示，不因按钮存在而随意换行；
- 字段宽度优先通过 Grid/Flex 自适应，避免硬编码零散宽度；
- 查询区域支持 Enter 触发查询；
- 宗族变化后清空失效的支派、字辈、代次和结果数据。

### 3.3 更多筛选

基础条件右侧提供“更多筛选”文字按钮。

交互规则：

- 默认收起；
- 点击后在基础条件下方展开第二行；
- 展开后按钮文字变为“收起筛选”；
- 展开、收起不清空已选择条件；
- 基础条件和扩展条件之间不增加横线、嵌套 Card 或独立边框；
- 通过约 `16 px` 的垂直间距表达分组。

### 3.4 扩展查询条件

人物档案标准扩展字段：

| 顺序 | 字段 | 控件 | 选择方式 |
|---|---|---|---|
| 1 | 档案状态 | 多选下拉框 | 支持全选、清空 |
| 2 | 性别 | 多选下拉框 | 支持全选、清空 |
| 3 | 字辈 | 可搜索多选下拉框 | 支持全选、搜索、清空 |
| 4 | 代次 | 多选下拉框 | 支持全选、清空 |

多选下拉统一要求：

- 使用 Ant Design `Select mode="multiple"`，不得用大面积平铺 Checkbox 代替；
- 提供“全选/取消全选”能力；
- 支持一键清空当前字段；
- 选项过多时支持搜索；
- 已选项较多时使用 `maxTagCount="responsive"` 或“已选 N 项”；
- 下拉项显示业务名称，不展示接口状态码或数据库 ID；
- “全选”是前端便捷操作，不向后端提交特殊伪值；
- 提交查询时传递真实选项数组。

档案状态建议值：

```text
草稿、待审核、正式、已驳回、已归档
```

性别建议值：

```text
男、女、未知
```

### 3.5 查询操作

查询操作固定为：

```text
更多筛选    重置    查询
```

按钮层级：

- “更多筛选”：`type="link"` 或 `type="text"`；
- “重置”：默认按钮；
- “查询”：唯一主按钮；
- 同组按钮间距统一为 `8 px`；
- 异步查询使用 `loading`，并禁止重复提交。

桌面端优先与第一行条件同一视觉行靠右对齐；宽度不足时整体换到下一行右侧，不拆散按钮组。

### 3.6 重置行为

默认重置规则：

- 保留当前宗族；
- 支派恢复为“全部”或业务默认值；
- 清空姓名、关键词和所有扩展筛选；
- 页码恢复为第 1 页；
- 排序恢复默认值；
- 收起更多筛选；
- 清除错误、批量选择和旧定位状态；
- 是否自动执行默认查询由页面产品约定决定，项目内同类页面保持一致。

## 4. 结果 Card 与页面级操作

### 4.1 标题与总数

结果 Card 标题统一为：

```text
查询结果
```

总记录数不拼接进标题，放在结果说明或分页区域：

```text
共 128 条记录
```

Card 标题用于表达区域任务，总数属于运行时状态，两者不耦合。

### 4.2 页面级业务操作位置

页面级业务操作放在结果 Card 标题右侧，与标题同一行，不放在查询 Card、表格操作列或页面外重复区域。

人物档案标准操作：

```text
查询结果                           创建人物  导出人物
```

操作排序：

```text
新增类 → 导入类 → 导出类 → 其他低频操作
```

同一操作区默认直接展示不超过 3 个按钮，超过后把低频动作收入“更多”。

### 4.3 主次按钮

同一个页面级操作区只允许一个主按钮。

| 操作 | 按钮类型 | 说明 |
|---|---|---|
| 创建人物 | 主按钮 | 页面核心新增任务 |
| 导出人物 | 默认/描边按钮 | 数据输出辅助任务 |
| 批量导入 | 默认按钮 | 非最高频写入操作 |
| 更多 | 文字或图标按钮 | 收纳低频操作 |

禁止“创建人物”和“导出人物”同时使用主按钮样式。

### 4.4 按钮命名

按钮文案使用“动词 + 业务对象”，建议 2～6 个汉字。

推荐：

- 创建人物；
- 导出人物；
- 导入资料；
- 批量归档。

禁止：

- 创建；
- 新建；
- 导出；
- 操作；
- 更多功能。

同一系统中同一动作必须使用同一名称，不混用“新增人物 / 创建人物”“导出数据 / 下载人物”。

### 4.5 图标

- 高频页面级操作可采用“图标 + 文字”；
- 图标位于文字左侧；
- 创建使用加号图标，导出使用导出/下载图标；
- 不得仅用无文字图标表达“创建人物”“导出人物”；
- 纯图标按钮必须提供 Tooltip 和 `aria-label`。

## 5. 创建类操作规范

以“创建人物”为例。

### 5.1 展示与权限

- 用户具备创建权限时展示；
- 完全无权限且无法申请时隐藏；
- 用户有权限但前置条件不满足时禁用并说明原因；
- 前端展示不是安全边界，后端必须再次鉴权。

### 5.2 前置条件

创建人物前至少校验：

1. 已选择宗族；
2. 已选择支派；
3. 用户具备当前宗族数据范围内的创建权限；
4. 当前宗族和支派处于可编辑状态。

提示示例：

```text
请先选择宗族后再创建人物。
请先选择支派后再创建人物。
```

### 5.3 跳转与上下文

点击创建后：

- 进入人物创建页面或现有人物录入向导；
- 自动带入当前宗族和支派；
- 不携带当前列表人物 ID；
- 保存成功后可以返回原查询页；
- 返回后恢复查询条件、排序、页码和滚动位置；
- 新建对象满足当前条件时，刷新后应出现在结果中。

### 5.4 提交状态

- 跳转或提交期间防重复点击；
- 保存按钮使用 `loading`；
- 请求失败保留用户输入；
- 创建成功使用轻量成功提示：`人物创建成功`。

---

## 6. 导出类操作规范

以“导出人物”为例。

### 6.1 默认范围

默认导出“当前查询条件匹配的全部结果”，不是仅导出当前页。

导出请求必须携带全部有效条件：

- 宗族；
- 支派；
- 姓名；
- 关键词；
- 档案状态；
- 性别；
- 字辈；
- 代次；
- 排序；
- 数据权限上下文。

只有按钮明确命名为“导出当前页”时，才允许仅导出当前页数据。

### 6.2 可用状态

以下情况禁用导出：

- 尚未执行查询；
- 查询结果为 0；
- 当前查询失败且无可用结果；
- 正在导出；
- 用户无导出权限。

禁用原因必须可发现，例如 Tooltip：

```text
当前没有可导出的查询结果。
```

### 6.3 大数据量导出

小数据量可同步下载；大数据量必须创建异步导出任务：

1. 提交导出条件；
2. 返回导出任务 ID；
3. 提示任务已创建；
4. 后台生成文件；
5. 在通知或导出记录中提供下载入口。

提示示例：

```text
导出任务已创建，文件生成后可在导出记录中下载。
```

### 6.4 敏感数据与权限

- 查询权限和导出权限必须分离；
- 导出字段按用户权限过滤或脱敏；
- 敏感、大批量导出建议二次确认；
- 确认信息至少包含导出范围、记录数、文件格式和敏感字段说明；
- 后端必须记录导出审计日志。

---

## 7. 批量操作规范

批量操作只在选中记录后展示或启用，放在表格上方的批量操作栏，不与“创建人物、导出人物”混放。

```text
已选择 5 项    批量归档    批量调整状态    取消选择
```

要求：

- 明确显示选中数量；
- 默认不跨页保留选择，若跨页保留必须明确说明；
- 危险批量操作二次确认；
- 部分成功必须展示成功数、失败数和失败原因；
- 完成后清空选择；
- 批量操作不得通过逐条前端循环代替后端批量接口。

## 8. 表格规范

### 8.1 人物档案标准列

| 顺序 | 字段 |
|---|---|
| 1 | 姓名 |
| 2 | 别名 |
| 3 | 性别 |
| 4 | 字辈 |
| 5 | 代次 |
| 6 | 支派 |
| 7 | 生卒 |
| 8 | 是否在世 |
| 9 | 配偶 |
| 10 | 档案状态 |
| 11 | 操作 |

字段按“身份信息 → 宗族信息 → 生命周期 → 状态与操作”排序。

### 8.2 字段展示

- 姓名是最左侧主识别字段，使用可聚焦链接进入详情；
- 空值统一显示 `-`，姓名为空显示“未命名人物”；
- 多个别名、配偶使用顿号分隔；
- 性别显示“男 / 女 / 未知”，不得只使用图标；
- 代次统一显示“第 N 世”或“N 世”；
- 支派显示业务名称，不显示 ID，超长时省略并提供 Tooltip；
- 生卒格式统一为 `1950-2020`、`1950-`、`?-2020` 或 `-`；
- 是否在世显示“是 / 否 / 未知”；
- 状态使用“颜色 + 文字”标签，不能只依赖颜色。

状态颜色建议：

| 状态 | 颜色语义 |
|---|---|
| 草稿 | 默认/灰色 |
| 待审核 | 蓝色 |
| 正式 | 绿色 |
| 已驳回 | 红色 |
| 已归档 | 灰色 |

### 8.3 操作列

操作列固定在最右侧：

```text
查看    编辑    更多
```

要求：

- 操作顺序全局一致；
- 查看、编辑使用文字按钮；
- 低频、危险操作进入“更多”；
- 操作列宽度固定；
- 点击操作不得触发行点击；
- 无权限操作隐藏；
- 危险操作二次确认；
- 横向滚动后操作列始终可见。

可根据业务需要固定姓名列在左侧，但操作列必须固定。

### 8.4 行交互

- 推荐保留姓名链接作为主要入口；
- 支持整行点击时，姓名和操作仍须可键盘聚焦；
- 行悬浮提供轻量高亮；
- 行点击、名称点击和查看操作的目标保持一致；
- 不允许整行点击成为唯一入口。

## 9. 分页、排序和 URL 状态

分页区域放在结果底部，包含：

- 总记录数；
- 当前页码；
- 页码切换；
- 每页条数；
- 数据量较大时可提供快速跳页。

示例：

```text
共 128 条    10 条/页                       < 1 2 3 … 13 >
```

规则：

- 查询条件变化后恢复第 1 页；
- 分页、排序、基础条件和扩展条件同步到 URL；
- 刷新、分享、前进后退必须恢复页面状态；
- 进入详情/编辑后返回列表，恢复条件、页码、排序和滚动位置；
- 多选查询参数使用稳定格式，例如逗号分隔或重复参数，项目内统一。

## 10. 页面状态

### 10.1 初始状态

未执行查询时显示：

```text
请设置查询条件后开始查询。
```

不得用“暂无数据”冒充尚未查询。

### 10.2 加载状态

- 查询按钮和表格显示 loading；
- 刷新时保留上一次成功结果，不闪空；
- 防止重复请求；
- 最新请求覆盖旧请求，旧响应不得回写当前页面。

### 10.3 空结果

```text
未找到符合当前条件的数据，请调整筛选条件。
```

可提供“重置条件”快捷操作。

### 10.4 查询失败

区分：

- 首次查询失败：结果 Card 内展示错误状态和重试；
- 刷新失败：保留旧结果，并提示“当前仍展示上一次成功结果”；
- 权限不足：展示 403，不泄露数量、名称或摘要；
- 局部筛选项加载失败：就近 Alert，不阻断仍可使用的条件。

### 10.5 操作反馈

- 查询成功不弹成功消息；
- 创建、导出、批量操作成功使用轻量消息；
- 操作失败保留现场并提供重试；
- 部分成功不得只提示“操作完成”。

---

## 11. 响应式规范

### 11.1 桌面端

- 基础条件默认一行；
- 扩展条件第二行；
- 页面级操作在结果标题右侧；
- 使用完整表格；
- 操作列固定右侧。

### 11.2 中等宽度

- 查询条件允许按完整字段组换行；
- 按钮组整体靠右，不拆散；
- 表格横向滚动；
- 操作列保持固定。

### 11.3 移动端

- 查询条件单列排列；
- 更多筛选纵向展开；
- 查询、重置保持明确主次；
- 页面级按钮移动到结果标题下方，顺序不变；
- 结果优先转换为 Card List；
- 每张卡片至少展示名称、支派、代次、状态和高频操作；
- 点击热区不小于 `44 × 44 px`。

---

## 12. 视觉与可访问性

- 页面背景浅灰，业务 Card 白色；
- 两个 Card 使用统一圆角、边框和阴影；
- 查询标题和结果标题层级一致；
- 不在基础条件和扩展条件之间使用横线；
- 通过间距而不是多层 Card 表达分组；
- 同一按钮组间距 `8 px`；
- 状态颜色仅作辅助，必须同时显示文字；
- 所有按钮支持键盘聚焦和 Enter/Space；
- 图标按钮提供 `aria-label`；
- 异步区域使用 `aria-busy`；
- 禁用原因不得只依赖颜色。

## 13. Ant Design 推荐实现

### 13.1 查询区域

```tsx
<Card title="人物档案查询">
  <Form layout="vertical" onFinish={handleSearch}>
    <div className="query-grid query-grid--primary">
      <Form.Item label="宗族">...</Form.Item>
      <Form.Item label="支派">...</Form.Item>
      <Form.Item label="姓名">...</Form.Item>
      <Form.Item label="关键词">...</Form.Item>
      <div className="query-actions">
        <Button type="link" onClick={toggleAdvanced}>更多筛选</Button>
        <Button onClick={handleReset}>重置</Button>
        <Button type="primary" htmlType="submit" loading={querying}>查询</Button>
      </div>
    </div>

    {advancedOpen ? (
      <div className="query-grid query-grid--advanced">
        <Form.Item label="档案状态"><Select mode="multiple" ... /></Form.Item>
        <Form.Item label="性别"><Select mode="multiple" ... /></Form.Item>
        <Form.Item label="字辈"><Select mode="multiple" showSearch ... /></Form.Item>
        <Form.Item label="代次"><Select mode="multiple" ... /></Form.Item>
      </div>
    ) : null}
  </Form>
</Card>
```

## 14. 禁止事项

- 页面顶部增加无价值介绍 Card；
- 页面外标题与查询 Card 标题重复；
- 查询 Card 标题写“查询条件”；
- 高频查询条件默认拆成多行；
- 扩展条件使用大面积平铺 Checkbox；
- 基础条件与扩展条件之间增加横线；
- 创建、导出按钮放入查询条件操作区；
- 创建、导出同时使用主按钮；
- 页面级操作、批量操作和行级操作混放；
- 结果 Card 标题拼接实时总数；
- 操作列不固定；
- 导出默认只导出当前页；
- 查询失败后清空上次成功结果；
- 状态只用颜色表达；
- 重置后保留失效联动条件；
- 无权限数据仍展示数量或摘要；
- 通过逐条前端请求实现批量操作。

## 16. 实现验收清单

### 16.1 查询区

- [ ] 查询标题位于查询 Card 内；
- [ ] 页面外无重复业务标题或介绍 Card；
- [ ] 基础条件默认只有一行；
- [ ] 基础条件包含宗族、支派、姓名、关键词；
- [ ] 更多筛选展开第二行且无横线分隔；
- [ ] 档案状态、性别、字辈、代次均为多选下拉；
- [ ] 多选字段支持全选、清空，必要时支持搜索；
- [ ] 更多筛选、重置、查询的位置和主次正确；
- [ ] 查询条件、分页和排序可由 URL 恢复；
- [ ] 宗族变化后正确清理失效条件。

### 16.2 结果区

- [ ] 结果 Card 标题为“查询结果”；
- [ ] 创建人物、导出人物位于标题右侧；
- [ ] 创建人物为唯一主按钮；
- [ ] 导出人物为次按钮；
- [ ] 表格字段顺序符合业务规范；
- [ ] 姓名是可聚焦的详情入口；
- [ ] 操作列固定右侧；
- [ ] 查看、编辑、更多顺序统一；
- [ ] 总数、分页和每页条数完整；
- [ ] 初始、加载、空结果、刷新失败、403 状态完整。

### 16.3 创建与导出

- [ ] 创建前校验宗族、支派和权限；
- [ ] 创建页面自动带入宗族、支派；
- [ ] 返回列表恢复查询现场；
- [ ] 导出默认覆盖当前条件下全部结果；
- [ ] 导出请求携带全部有效查询条件；
- [ ] 无结果或无权限时不可导出；
- [ ] 大数据量使用异步导出任务；
- [ ] 敏感字段按权限过滤并记录审计日志。

### 16.4 批量与行级操作

- [ ] 批量操作仅在选中数据后出现；
- [ ] 批量栏显示选中数量；
- [ ] 危险操作二次确认；
- [ ] 部分失败展示成功数、失败数和原因；
- [ ] 行级按钮不触发行点击；
- [ ] 无权限操作正确隐藏；
