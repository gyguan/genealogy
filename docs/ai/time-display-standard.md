# 时间展示规范

本项目所有面向用户的时间展示、进度汇报、PR 说明、任务看板、恢复检查点和操作结果，统一使用北京时间。

## 1. 标准时区

- 时区名称：`Asia/Shanghai`
- UTC 偏移：`UTC+8`
- 展示名称：北京时间
- 不使用日本时间、浏览器本地时间或未经说明的 UTC 时间作为默认展示时间。

## 2. 展示格式

推荐使用绝对时间：

```text
2026-07-13 16:30（北京时间）
```

需要精确到秒时：

```text
2026-07-13 16:30:25（北京时间）
```

日期存在歧义时，应使用完整年月日，不只使用“今天、明天、昨天”等相对表达。

## 3. 外部系统时间

GitHub、CI、日志或第三方接口返回 UTC 或其他时区时：

1. 面向用户汇报时先转换为北京时间；
2. 排查问题需要保留原始时间时，可同时展示原始时区；
3. 必须明确标注两个时间分别属于哪个时区。

示例：

```text
GitHub 原始时间：2026-07-13 08:51 UTC
北京时间：2026-07-13 16:51（UTC+8）
```

## 4. 适用范围

该规则适用于：

- 聊天中的进度和完成时间；
- Issue、PR、Commit 和 Review 说明；
- 任务看板和恢复检查点；
- CI、构建、测试和部署结果；
- 文档、截图说明和审计记录；
- 后续所有 AI 辅助研发任务。

除非用户明确指定其他时区，否则不得改用其他时区。
