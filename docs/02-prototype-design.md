# 02. 原型设计说明

## 原型目标

MVP 1 原型用于演示管理端核心闭环：工作台、宗族管理、支派管理、字辈管理、人物管理、关系维护、来源绑定、审核中心、世系图、导入导出、权限管理。

## 信息架构

```text
首页工作台
├── 数据概览
├── 待审核事项
└── 快捷入口

宗族管理
支派管理
字辈管理
人物管理
世系图
资料来源
审核中心
导入导出
权限管理
系统日志
```

## 核心页面

| 页面 | 说明 |
|---|---|
| 首页工作台 | 展示总人数、支派数、待审核、资料来源、最近修改 |
| 宗族管理 | 维护宗族名称、姓氏、堂号、郡望、始祖、简介 |
| 支派管理 | 左侧支派树，右侧支派详情和成员统计 |
| 字辈管理 | 维护派语、世次映射和校验规则 |
| 人物列表 | 支持按姓名、支派、世次、字辈、状态筛选 |
| 人物详情 | 展示基础档案、亲属关系、传记、来源和修订记录 |
| 关系编辑 | 弹窗维护父母、配偶、子女、继嗣、出嗣等关系 |
| 世系图 | 支持从某人查看家庭图、上溯图和下延树 |
| 资料来源 | 上传资料，绑定人物或关系 |
| 审核中心 | 待审核列表、变更对比、通过、驳回 |
| 导入导出 | 上传 Excel、校验错误、确认导入草稿 |
| 权限管理 | 邀请成员、分配角色、设置支派范围 |

## 原型入口

```text
prototype/index.html
```

直接用浏览器打开即可演示。

## 交互原则

1. 人物录入要高效，支持快速添加父母、配偶、子女。
2. 正式数据不能直接改，必须生成变更草稿。
3. 世系图默认展示有限代数，支持展开和支派过滤。
4. 资料来源与人物、关系关联，提升数据可信度。
