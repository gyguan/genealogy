# Genealogy 前端页面规范

## 1. 目标与范围

针对当前页面在标题、查询、列表、详情、表单、工作台、流程操作和反馈方式上的差异，建立一套可以直接复用、评审和验收的中后台页面模式。

本规范优先参考 Ant Design 的设计价值观和组件模式，并结合 Genealogy 当前 React + TypeScript + Ant Design 实现落地。

规范原型入口：

```text
/page-patterns.html
```

原型为独立入口，不加入正式系统菜单，也不请求真实业务 API。规范覆盖以下层次：

1. 全局尺寸、按钮、Tabs、状态和响应式基线；
2. 查询列表、详情、编辑等标准 CRUD 页面；
3. 看板、向导、画布、导入、任务审核等专业工作台页面；
4. 权限、审计、附件、安全登录等专项页面；
5. 现有业务模块到目标页面模式的映射和迁移顺序。

已经确认的全局决策：

- 查询区与列表区采用双 Card；
- 复杂编辑采用独立页面；
- 普通对象详情在桌面端统一采用 `720 px` Drawer，移动端占满视口。

## 2. 设计原则

### 2.1 确定性

- 同一页面类型使用相同结构、操作位置和状态反馈。
- 相同状态使用相同颜色、图标、文案和组件。
- 同类对象的 Tab 名称、顺序、字段分组和动作名称保持一致。

### 2.2 信息分层

页面信息按以下层级组织：

1. 页面上下文：面包屑、标题、说明、宗族或对象范围；
2. 核心任务：新增、提交、审核、导入等唯一主操作；
3. 业务内容：筛选、列表、详情、表单、图表或画布；
4. 辅助信息：规则提示、证据、追踪、帮助说明；
5. 异常与反馈：就近展示，不覆盖主要内容。

### 2.3 直接与克制

- 一个页面或弹窗只保留一个视觉主操作。
- 常用筛选直接展示，低频条件进入“更多筛选”。
- 表格行内只展示高频动作，其余进入“更多”。
- 页面过长不等于必须拆成 Tabs，应先判断业务分区是否并列且相对独立。
- 不向业务用户暴露数据库 ID、接口字段名、技术错误堆栈。

### 2.4 即时反馈与状态隔离

- 表单错误显示在字段附近。
- 查询错误显示在结果区域内。
- 操作成功使用轻量 `message.success`。
- 操作失败保持弹窗、表单或当前工作现场，不丢失输入。
- 一个 Tab、图表或子面板失败，不应阻断页面其他可用内容。
- 禁止通过未处理 Promise 或抛异常代替表单校验。

### 2.5 页面类型优先

设计和开发页面前，先从本规范的页面模式目录中选择最接近的类型，不允许先自由拼装组件，再以“业务特殊”为由形成新的页面结构。

## 3. 全局设计基线

| 项目 | 桌面端 | 移动端 | 说明 |
|---|---:|---:|---|
| 页面内容外边距 | 24 px | 12 px | 应用外壳统一控制 |
| 页面模块间距 | 16 px | 12 px | 页面头、Card 和工作区之间 |
| Card 内容边距 | 16 px | 16 px | 页面头可使用 20～24 px |
| 表单项间距 | 12 px | 12 px | 通过主题统一配置 |
| 标准控件高度 | 32 px | 40 px | 桌面默认 `middle` |
| 触控热区 | 不强制 | ≥ 44 × 44 px | 可通过容器补足 |
| 页面内容最大宽度 | 1600 px | 100% | 画布工作台可突破此限制 |
| 标准圆角 | 8 px | 8 px | 大 Card 可使用 12 px |
| 普通详情抽屉 | 720 px | 100vw | 查看型普通业务对象 |
| 画布检查器 | 480～560 px | 100vw | 避免遮挡主画布 |
| 复杂资源详情 | 独立页面 | 独立页面 | 不使用 900 px 以上抽屉 |
| 查询列表表格 | middle | small/middle | 移动端优先 Card List |

### 3.1 页面主操作

- 页面主操作位于页面头右上角。
- 每个页面只允许一个 `type="primary"` 的页面级主操作。
- 宗族选择器、时间范围和视图切换属于上下文，不属于主操作。
- 删除、撤销、高风险授权等危险操作不得使用主色按钮。
- 返回操作优先放在页面头左侧或面包屑中，不与右侧操作混排。

### 3.2 标题层级

| 层级 | 用途 | 推荐实现 |
|---|---|---|
| 页面标题 | 页面任务名称 | `Typography.Title level={3}` |
| Card 标题 | 查询区、列表区、信息分组 | `Card title` |
| 详情标题 | 当前对象名称 | `Typography.Title level={4}` |
| 内容小节 | 摘要、正文、附件等 | `Title level={5}` 或小 Card |

### 3.3 按钮规范

#### 3.3.1 尺寸与点击热区

优先使用 Ant Design 内置的 `small`、`middle`、`large`，不在业务页面自行定义零散高度。

| 按钮类型 | 推荐尺寸 | 最小宽度 | 使用场景 |
|---|---:|---:|---|
| 页面主按钮 | 32 px；重点任务可 40 px | 88 px | 新增、创建、提交审核、确认发布 |
| 普通操作按钮 | 32 px | 72 px | 查询、重置、保存草稿、取消 |
| 表格文字按钮 | 32 px 热区 | 不固定 | 查看、编辑等行内动作 |
| 纯图标按钮 | 32 × 32 px | 32 px | 更多、刷新、关闭、缩放 |
| 移动端按钮 | 40 px | 88 px 或等宽 | 页面头和底部操作区 |

- 触屏可点击热区不得小于 `44 × 44 px`。
- 同一操作组中的按钮高度一致。
- 不允许把普通按钮压缩到 28 px、30 px 等非标准尺寸。
- 长文案优先调整布局或移入“更多”，不得缩小字号强行容纳。

#### 3.3.2 语义层级

| 层级 | Ant Design 实现 | 使用规则 |
|---|---|---|
| 主操作 | `type="primary"` | 一个页面或弹窗最多一个 |
| 次要操作 | 默认按钮 | 保存草稿、重置、返回、取消 |
| 轻量操作 | `type="link"` / `type="text"` | 查看、编辑、帮助、展开 |
| 危险操作 | `danger` | 删除、撤销、不可逆操作 |
| 不可用操作 | `disabled` | 状态暂时不可执行且用户需要知道其存在 |

- “保存草稿”和“提交审核”同时存在时，提交审核为主操作。
- 无权限且无法申请恢复的操作直接隐藏；状态性不可用才禁用并说明原因。
- 危险操作默认进入“更多”或独立确认弹窗，不与主操作紧邻。

#### 3.3.3 位置与顺序

统一遵循“次要在左、主要在右”。

| 页面区域 | 统一位置 | 推荐顺序 |
|---|---|---|
| 页面头 | 右上角 | 上下文、次要操作、主操作 |
| 查询 Card | 右下角 | 重置、查询 |
| 列表工具栏 | Card 标题右侧 | 批量操作、视图设置、刷新 |
| 表格操作列 | 最右侧 | 查看、编辑、更多 |
| 详情 Drawer Header | 标题右侧 | 追踪、编辑、提交审核、更多 |
| 编辑页固定栏 | 右侧 | 取消、保存草稿、提交审核 |
| Modal Footer | 右侧 | 取消、确认 |
| 空态/错误态 | 说明下方居中 | 次要操作、主操作 |

- 同一页面不得在 Header、正文末尾和底部重复同一组操作。
- 列表页“新增”只放页面头，首次空态可提供一次引导入口。
- Modal 确认按钮始终在最右侧。

#### 3.3.4 间距、图标和状态

- 同一按钮组间距为 `8 px`；纯图标按钮间可使用 `4 px`。
- 不同语义组间隔 `16 px`，危险操作与普通操作至少间隔 `16 px`。
- 表格的查看、编辑优先使用文字按钮；纯图标按钮必须提供 Tooltip 和 `aria-label`。
- 异步操作使用 `loading`，保持原宽度并防止重复提交。
- 请求失败后恢复按钮可用状态，并保留输入和上下文。
- 按钮文案使用动宾短语，建议 2～6 个汉字，同一动作全局一致。

### 3.4 页面内容分区与 Tabs

Tabs 用于同一页面上下文中多个并列、相对独立且需要频繁切换的内容区域，不能仅因页面较长就机械拆分。

#### 3.4.1 组件选择

| 内容特征 | 推荐模式 |
|---|---|
| 1～2 个短分组 | Card 或锚点 |
| 3～5 个并列分区 | Tabs |
| 6～7 个稳定分区 | Tabs + 重新归类或“更多” |
| 超过 7 个分区 | 二级导航或独立子页面 |
| 存在明确先后顺序 | Steps / 分步表单 |
| 不同对象或不同核心任务 | 路由、菜单或独立页面 |
| 展开少量补充内容 | Collapse |

满足以下条件时使用 Tabs：

- 共享同一页面标题、业务对象和权限上下文；
- 分区是并列关系，不是先后步骤；
- 用户需要频繁切换或对照；
- 每个分区可独立加载和处理错误；
- 连续滚动会明显降低定位效率。

以下情况不使用 Tabs：

- 只有一个有效分区；
- 用 Tabs 掩盖字段分组混乱；
- 用户必须按顺序完成任务；
- 分区属于不同权限域、对象或页面主任务；
- 需要嵌套两层及以上 Tabs。原则上禁止嵌套 Tabs。

#### 3.4.2 结构、命名与状态

```text
页面头：标题、对象范围、页面级主操作
对象摘要或全局提示（可选）
Tabs 导航
当前 Tab 内容：工具栏、数据、局部状态
```

- 页面标题、对象状态、全局摘要和页面级操作放在 Tabs 外。
- 推荐 `2～5` 个 Tab，桌面端直接展示不得超过 `7` 个。
- Tab 名称使用 2～6 个汉字的稳定业务名词。
- 顺序保持“基本信息 → 高频业务 → 来源附件 → 审核追踪”。
- 数量徽标只用于有业务价值的待办数、未读数或对象数，不默认显示 0。
- 页面级操作不在每个 Tab 重复；Tab 专属操作放当前内容区右上角。

#### 3.4.3 URL、加载与表单

- 激活 Tab 通过 `tab=<key>` 或领域专用参数写入 URL。
- 刷新、前进后退和分享链接应恢复当前 Tab。
- 每个列表 Tab 分别保留分页、排序、筛选和滚动位置。
- 默认只加载激活 Tab；单个 Tab 失败不得阻断其他内容。
- 表单、草稿等需要保持状态的 Tab 默认不销毁，或把状态提升到页面级。
- 跨 Tab 表单提交时校验全部 Tab，自动跳转到第一个错误 Tab。
- Steps 表达流程先后，Tabs 不得模拟向导。

#### 3.4.4 移动端

- 标签保持单行横向滚动，不允许自动换成两行。
- 首屏建议可见 3～4 个 Tab，低频项进入“更多”或下拉选择。
- 点击热区不小于 44 px，激活状态不能只依赖颜色。
- 禁止移动端嵌套 Tabs。

## 4. 页面模式目录与当前模块映射

### 4.1 页面模式目录

| 模式 | 主要任务 | 典型组件 |
|---|---|---|
| 查询列表页 | 找到对象并进入后续任务 | Search Card、Result Card、Table |
| 详情抽屉 | 保留列表现场查看普通对象 | Drawer、Descriptions、Tabs |
| 独立详情页 | 长期查看和处理复杂对象 | Page Header、Summary、Tabs、子表 |
| 简单编辑弹窗 | 快速修改少量属性 | Modal、Form |
| 复杂编辑页 | 分组编辑、草稿、审核提交 | Card Form、Sticky Actions |
| 数据看板 | 监控指标、趋势和下钻 | Statistic、Chart、Drawer |
| 分步向导 | 按顺序完成跨对象任务 | Steps、Form、Result |
| 可视化画布工作台 | 在图、地图或关系画布中分析 | Toolbar、Canvas、Inspector |
| 文件导入与批次任务 | 上传、预检、执行和修错 | Upload、Steps、Table、Progress |
| 任务与审核工作台 | 处理待办、风险和审核队列 | KPI、Filters、Table、Drawer |
| 成员与权限 | 配置角色和数据范围 | Table、Grant Modal、Audit |
| 审计与变更追踪 | 只读检索、时间轴和 Diff | Tabs、Timeline、Diff Drawer |
| 文档与附件资源库 | 管理文件、引用、预览和权限 | Upload、Preview、Binding Table |
| 登录与安全页面 | 登录、邀请和账号恢复 | Standalone Form、Alert |

### 4.2 当前模块映射

| 当前模块 | 主页面模式 | 辅助模式 | 目标要求 |
|---|---|---|---|
| 族谱首页 | 数据看板 | 指标下钻抽屉 | 采用看板专项规范 |
| 建谱向导 | 分步向导 | 结果页、审核入口 | 采用 Steps，不使用 Tabs 表达流程 |
| 人物档案 | 查询列表、独立详情 | 复杂编辑页 | 复杂编辑退出 Drawer |
| 世系图谱 | 可视化画布工作台 | 画布检查器 | 允许 480～560 px 检查器例外 |
| 来源资料库 | 查询列表、独立详情 | 附件和引用管理 | 920 px Drawer 迁移为独立详情页 |
| 宗族文化 | 查询列表、详情抽屉 | 复杂编辑页、Tabs | 作为标准 CRUD 基准实现 |
| 数据导入 | 文件导入与批次任务 | Steps、任务管理 | 统一预检、进度和失败处理 |
| 修谱工作台 | 任务工作台 | 指标摘要、任务详情 | KPI、筛选、任务池统一结构 |
| 审核中心 | 审核工作台 | 批量操作、时间轴 | 统一批量决策和部分失败反馈 |
| 成员与权限 | 成员与权限 | 详情抽屉、授权弹窗 | 强化范围预览和高风险确认 |
| 审计追踪 | 审计与变更追踪 | 导出、Diff、时间轴 | 明确只读和敏感信息规则 |
| 登录与邀请 | 登录与安全页面 | 找回密码、接受邀请 | 独立安全页面规范 |

### 4.3 组合原则

- 一个页面只能有一个“主模式”，其他模式作为其子区域或动作。
- 看板下钻可以使用 Drawer，但不能把整个看板塞进普通列表页模板。
- 画布工作台可包含搜索面板，但不能按普通双 Card 查询列表布局压缩画布。
- 导入页可包含 Steps 和 Tabs：Steps 表达流程阶段，Tabs 只表达导入对象类型。
- 审核工作台可包含稳定视图 Tabs，但公共筛选、批量工具栏和详情结构必须统一。

## 5. 查询列表页规范

### 5.1 标准结构

```text
页面头：面包屑、标题、说明、范围、唯一主操作
查询 Card：常用筛选、更多筛选、重置、查询
结果 Card：标题和总数、工具栏、状态、Table/Card List、分页
```

### 5.2 查询区

- 使用 `Form layout="vertical"`。
- 桌面端每行建议 4～6 个筛选项，常用条件不超过 8 个。
- 查询和重置位于右下角，顺序固定为“重置、查询”。
- 查询成功不弹成功消息；查询失败在结果 Card 内展示 Alert 或 Result。
- 公共上下文如宗族选择放页面头，不在每个查询 Card 重复。
- 筛选、分页和排序应可通过 URL 恢复。

### 5.3 列表区

- Card 标题使用“对象名称 + 总数”。
- 名称列放在最左侧，并承担进入详情的主入口。
- 操作列固定在右侧，默认不超过“查看、编辑、更多”。
- 审核、归档、删除等低频动作进入“更多”或详情。
- 不建议整行作为唯一入口；支持整行点击时仍保留可聚焦的名称链接。
- 批量操作只在选中数据后出现，且明确作用范围和选中数量。

### 5.4 状态

- 首次加载：Skeleton 或表格 loading，不以 0 或空列表冒充结果。
- 空态：说明为什么无数据、下一步是什么、能否重置或新增。
- 首次失败：错误 Result/Alert，并提供重新加载。
- 刷新失败：保留上次成功结果，提示当前为旧数据。
- 无权限：使用 403，不展示受限对象名称、数量或摘要。

## 6. 详情模式规范

### 6.1 普通详情抽屉

满足以下条件时使用 `720 px` Drawer：

- 用户需要保留列表上下文；
- 详情以查看为主；
- 字段不超过约 18 个；
- 子任务较少；
- 不需要长期 URL 工作入口。

标准结构：

```text
Header：对象名称、状态、编号、更新时间、高频操作（最多 3 个）
Tabs：基本信息、来源附件、审核追踪
Content：Descriptions、摘要、子对象列表、局部状态
```

- 高频操作放 Header，低频和危险操作进入“更多”。
- 正文末尾不重复 Header 操作。
- 加载、404、403、服务失败和局部 Tracking 失败必须区分。

### 6.2 独立详情页

以下情况升级为独立详情页：

- 存在复杂编辑、审核、协作或长期处理任务；
- 多个大型子表或附件预览；
- 内容过长，需要大面积可视化；
- 需要稳定 URL、面包屑和浏览器历史；
- 预计 Drawer 宽度超过 800 px。

标准结构：

```text
页面头：返回、面包屑、对象名称、状态、页面级操作
对象摘要：关键属性、完整度、风险、更新时间
Tabs/锚点：基本信息、业务明细、来源附件、审核追踪
主内容区
可选右栏：负责人、权限、关联对象、最近活动
```

- 返回列表时恢复原筛选、分页和滚动位置。
- 页面级操作固定在页面头；Tab 专属操作放当前内容区。
- 长附件预览、Diff 或大型子表不得挤入普通 Drawer。

### 6.3 画布检查器例外

画布中选择节点、连线或地图点位时，使用 `480～560 px` 侧边检查器：

- 检查器是画布的辅助区域，不等同普通详情抽屉；
- 必须尽量保留可见画布，支持关闭和重新定位；
- 复杂编辑或完整档案通过按钮跳转独立页面；
- 移动端占满视口。

## 7. 编辑表单规范

### 7.1 简单编辑 Modal 与复杂编辑页

| 条件 | Modal | 独立编辑页 |
|---|---:|---:|
| 字段 ≤ 12 且逻辑简单 | 推荐 | 可选 |
| 字段 > 12 | 不推荐 | 推荐 |
| 需要多个业务分组 | 不推荐 | 推荐 |
| 存在长正文或附件 | 不推荐 | 推荐 |
| 多步骤任务 | 不适用 | 使用 Steps |
| 快速修改少量属性 | 推荐 | 不推荐 |

简单编辑 Modal：

- 推荐宽度 `520～640 px`；
- 一般使用单列，少量短字段可两列；
- Footer 固定为“取消、保存/确认”；
- 保存中禁止关闭和重复提交；
- 服务端错误保留表单并在弹窗内展示。

复杂编辑页：

```text
页面头：返回、标题、说明
状态 Alert：正式对象规则、驳回原因、并发提示
业务分组 Card 或 Tabs
底部固定栏：取消、保存草稿、提交审核
```

### 7.2 表单布局和校验

- 桌面端最多 3 列，长文本始终独占整行。
- 字段按业务语义分组，不按接口结构排列。
- 安全等级、可见范围、可信度统一放“治理与展示”。
- 关联对象使用业务搜索选择器，不要求输入数据库 ID。
- 使用 `Form.Item rules` 就地校验；校验失败不显示全局错误。
- 请求失败保留输入；未保存修改离开页面需要确认。
- 跨 Tab 表单提交校验全部 Tab，并定位首个错误字段。

### 7.3 底部操作区

```text
取消 | 保存草稿 | 提交审核
```

- 提交审核为主操作；保存草稿为默认按钮。
- 取消不触发保存。
- 操作栏固定时不得遮挡最后一个表单项。

## 8. 数据看板与指标下钻规范

适用于族谱首页、运营概览和质量监控页面。

### 8.1 标准结构

```text
页面头：标题、说明、全局范围、更新时间
核心 KPI：4～6 个
趋势分析
结构分布
待办与风险
最近活动
指标下钻 Drawer 或目标页面
```

### 8.2 KPI Card

- 第一屏核心 KPI 建议 4～6 个，次要指标进入后续分组。
- Card 必须包含指标名称、值、单位和必要的业务说明。
- 有同比或趋势时明确比较区间，不能只显示无语境的百分比。
- 可点击指标使用 hover、指针和“查看明细”等一致提示；不可点击指标不得伪装成入口。
- 加载状态使用 Skeleton 或 `-`，不得先显示 0。
- 风险数量使用语义色，但不能只依赖颜色表达状态。

### 8.3 图表和下钻

- 图表标题说明统计范围、时间范围和口径。
- 颜色、图例和 Tooltip 在同类图表中保持一致。
- 局部图表失败只影响当前 Card，其他指标继续展示。
- 无数据必须区分“确实为 0”和“尚未维护/加载失败”。
- 简单明细使用 720 px 下钻 Drawer；复杂分析跳转查询列表或独立详情页。
- 下钻条件应写入 URL 或明确展示在抽屉标题中。

## 9. 分步向导规范

适用于建谱向导、批量配置和有明确先后依赖的任务。

### 9.1 适用边界

使用 Steps 的条件：

- 用户必须按顺序或受前置条件约束完成任务；
- 每一步有明确输入、完成判定和后续影响；
- 流程可中断并需要恢复进度。

并列内容使用 Tabs，不使用 Steps；单页短表单不强行拆步骤。

### 9.2 标准结构

```text
页面头：任务名称、说明、退出入口
进度 Card：步骤、当前状态、简短说明
当前步骤内容
结果/错误提示
固定操作栏：上一步、保存草稿、下一步/提交
```

### 9.3 步骤状态和跳转

- 区分等待、进行中、已完成、存在错误、审核中和已驳回。
- 每一步定义完成判定，不能仅以“用户访问过”标记完成。
- 默认不允许跳过未满足的前置步骤；允许跳转时需说明后续影响。
- 前置数据变更导致后续失效时，必须明确标记并要求重新确认。
- 当前步骤写入 URL，例如 `step=person`；刷新后恢复当前步骤和草稿。
- 中途离开提供保存草稿或离开确认。

### 9.4 操作和移动端

- 操作顺序为“上一步、保存草稿、下一步/提交”。
- 最后一步主按钮使用“提交审核”“完成建谱”等明确文案，不使用“下一步”。
- 移动端可使用垂直 Steps 或“步骤 3/7 + 当前标题”的精简进度，不横向挤压全部步骤。

## 10. 可视化画布工作台规范

适用于世系图谱、迁徙地图、关系网络和其他大面积可视化页面。

### 10.1 标准结构

```text
页面头：标题、范围、导出/全屏等页面级操作
搜索与对象选择区
画布工具栏：模式、范围、深度、筛选、应用
画布：图例、加载/空态/错误、缩放与定位
右侧检查器：节点或连线详情
```

### 10.2 查询和工具栏

- 页面级搜索与画布内定位分开，避免用户混淆“重新查询”和“仅定位当前图”。
- 查询条件修改后显示“条件已修改”，应用成功后显示“条件已应用”。
- 模式切换在选项较少且互斥时使用 Segmented；复杂模式使用 Select 或路由。
- 工具栏只保留当前画布所需操作，低频项进入“更多设置”。
- 查询和渲染请求应取消旧请求或忽略过期响应。

### 10.3 画布交互

- 画布提供缩放、适配视口、回到中心、重置和必要图例。
- 节点、连线的默认、悬停、选中、路径高亮和受保护状态必须区分。
- 空态说明需要选择什么对象或调整什么范围。
- 超大数据时提示截断、聚合或深度限制，不静默丢弃节点。
- 渲染失败提供重试和可访问的列表替代信息。
- 键盘用户可以聚焦主要工具，并通过列表或检查器读取当前选中对象。

### 10.4 检查器

- 桌面端宽度 `480～560 px`；移动端占满视口。
- Header 显示对象类型、名称和关闭入口。
- “设为中心”“查看完整档案”等操作不超过 3 个，其余进入更多。
- 检查器只承担快速查看和导航，复杂编辑跳转独立页面。

## 11. 文件导入与异步批次规范

适用于人物、关系、来源导入，以及未来的大文件处理和批量生成任务。

### 11.1 标准流程

```text
1. 选择导入对象和目标范围
2. 下载模板或查看字段说明
3. 上传文件
4. 文件解析与预检
5. 处理错误、警告和疑似重复
6. 创建导入批次
7. 后台执行进度
8. 结果、失败明细和重试
```

### 11.2 上传和预检

- 明确支持的文件类型、大小、模板版本和编码。
- 上传前不自动提交，先展示文件名、大小和移除入口。
- 上传区区分空闲、上传中、解析中、预检完成和失败。
- 预检结果至少区分有效、警告、疑似重复和错误行。
- 是否允许带错误创建批次必须由业务规则明确，不能默认放行。
- 重复确认使用清晰复选框或确认步骤，不以普通提示消息代替。

### 11.3 批次执行

- 大批次进入后台执行，用户离开页面后任务继续运行。
- 状态至少包括排队中、执行中、暂停、重试等待、成功、部分成功、失败、取消。
- 展示总数、已处理、成功、失败、进度和最近更新时间。
- 支持下载失败明细、修正后重试或重新创建批次。
- 部分成功不能显示成完全成功；失败原因按批次和行级分别展示。
- 创建批次按钮使用 loading 并防止重复提交，服务端保证幂等。

### 11.4 导出和文件生成

- 小文件同步下载可使用按钮 loading。
- 预计超过数秒或结果较大时转为后台任务，并提供任务中心入口。
- 导出范围、字段、隐私规则和文件格式在执行前明确展示。
- 下载链接过期、生成失败和无权限必须有明确反馈。

## 12. 任务池与审核工作台规范

适用于修谱工作台、审核中心和其他需要连续处理待办的页面。

### 12.1 标准结构

```text
页面头：任务范围和页面级操作
公共筛选 Card
KPI 摘要：待处理、高风险、阻塞、超时等
稳定视图 Tabs（可选）
批量操作工具栏
任务列表
任务详情 Drawer
处理确认 Modal
```

### 12.2 KPI 与筛选

- KPI 仅在能稳定表达任务范围时使用；可点击 KPI 必须明确其会附加筛选。
- KPI 与筛选条件联动后，当前条件应可见并可一键清除。
- 公共筛选放 Tabs 外，不为每个视图复制一套表单。
- 视图 Tabs 用于“待我处理、我提交的、已处理”等稳定业务队列。

### 12.3 列表和单条处理

- 列表突出事项、对象、风险、状态、提交人、更新时间和建议动作。
- 行内默认展示“详情、主要处理、更多”；通过与驳回同时出现时避免两个高权重按钮并列。
- 推荐在详情 Drawer 或处理 Modal 中完成最终决策，列表行内操作作为快捷入口。
- 状态冲突时提示任务已被其他人处理，并刷新当前记录。

### 12.4 批量操作

- 批量工具栏只在选中记录后出现，显示选中数量和作用范围。
- 默认只选择当前页；如支持跨页选择，必须明确“已选择全部 N 条”。
- 批量通过、驳回等操作必须二次确认，说明可能的不同对象和结果。
- 返回部分成功、部分失败的结构化结果，并允许查看失败项。
- 批量完成后清空失效选择，保留当前筛选和分页。

### 12.5 任务详情和历史

- 普通任务详情使用 640～720 px Drawer，复杂协作任务升级独立页面。
- Header 显示风险、状态和高频操作；正文包含问题描述、影响范围、建议处理和关联入口。
- 审核历史使用 Timeline，展示轮次、提交人、审核人、时间、意见和耗时。
- 关联页面跳转前写入对象上下文，返回后恢复任务池现场。

## 13. 审核、归档、删除和高风险操作

### 13.1 统一结构

```text
弹窗标题
影响对象和当前状态
操作后果 Alert
操作原因或提交说明
取消 / 确认
```

| 操作 | 语义色 | 原因 | 确认文案 |
|---|---|---|---|
| 提交审核 | 主色/信息色 | 选填 | 确认提交 |
| 审核通过 | 主色 | 选填或按规则 | 确认通过 |
| 审核驳回 | 危险/警告 | 必填 | 确认驳回 |
| 归档草稿 | 警告 | 必填 | 确认归档 |
| 删除草稿 | 危险 | 按规则 | 确认删除 |
| 撤销授权 | 危险 | 必填 | 确认撤销 |
| 高风险授权 | 危险确认 | 必填 | 确认授权 |

- 成功：关闭弹窗、显示成功消息、刷新对象。
- 失败：弹窗保持打开，保留原因并展示服务端错误。
- 状态冲突：说明对象已被他人修改并提供刷新。
- 权限失败：不继续重试，隐藏后续不可用动作。

## 14. 专项页面规范

### 14.1 成员与权限

```text
成员列表
成员详情
当前授权列表
新增/编辑授权 Modal
角色能力说明
授权范围预览
高风险确认
权限变更历史
```

- 角色和数据范围必须同时展示，不只显示角色名称。
- 授权前展示实际可访问范围，支派范围说明是否包含下级支派。
- 高风险角色需要二次确认，说明能力和范围。
- 防止移除最后一个有效管理员，防止越级授权。
- 停用成员、移除成员和撤销授权是不同动作，文案和后果必须区分。
- 权限变更必须填写原因并进入审计历史。

### 14.2 审计、追踪和 Diff

- 审计页面默认只读，编辑按钮和主色操作必须克制。
- 时间轴展示对象创建、变更、审核、来源绑定和权限事件。
- 字段 Diff 展示字段业务名称、旧值、新值、操作者和时间。
- 敏感字段脱敏，技术字段默认折叠，不直接暴露请求体和堆栈。
- 403 与对象不存在必须区分；单段链路失败不阻断基本摘要。
- 导出入口只在真实权限存在时显示，并记录导出审计。

### 14.3 文档与附件资源库

- 支持列表视图；图片等媒体量较大时可增加缩略图视图。
- 上传状态、文件类型、大小、敏感级别、可见范围和引用数量必须展示。
- 图片、PDF、音视频等使用对应预览；不支持的格式提供下载兜底。
- 预览失败不影响下载；下载失败不伪装为无权限。
- 删除或替换前检查引用关系，受引用文件不得静默删除。
- 批量上传需要逐文件状态和部分成功结果。
- 多个大型预览、引用表和附件表应使用独立详情页，不使用超宽 Drawer。

### 14.4 登录、邀请和账号恢复

- 使用独立页面，不复用后台应用的侧边导航和页面头。
- 输入控件使用 large，主操作全宽，表单宽度保持适中。
- 登录和找回密码错误不得泄露账号是否存在。
- 密码规则、两次确认、凭据失效和会话失效应就近提示。
- 登录、重置和接受邀请均防止重复提交。
- 支持浏览器密码管理和正确的 `autocomplete`。
- 移动端避免键盘遮挡主按钮和错误提示。

## 15. 响应式与无障碍规范

### 15.1 断点策略

| 宽度 | 页面策略 |
|---|---|
| ≥ 1200 | 完整查询网格、Table、720 px Drawer、完整工作台 |
| 768～1199 | 查询减少列数、表格横向滚动、画布工具栏换行 |
| < 768 | 12 px 边距、单列筛选、Drawer 占满视口、优先 Card List |

### 15.2 移动端

- 横向表格只能作为过渡方案，核心页面逐步支持 Card List。
- 页面头直接展示不超过两个操作，其余进入“更多”。
- 长表单单列；底部操作可等宽但只突出一个主按钮。
- 看板 KPI 使用 1～2 列；图表提供可滚动或简化版本。
- 画布页面优先全屏，检查器占满视口。
- Steps 使用垂直或精简进度；Tabs 保持单行横向滚动。

### 15.3 无障碍

- 不把整行点击作为唯一入口，保留可聚焦链接或按钮。
- 纯图标按钮提供 Tooltip 和 `aria-label`。
- 状态、风险、错误和选中态不能只依赖颜色。
- 表单 Label、错误提示和输入控件建立正确关联。
- 图表和画布提供摘要、图例或列表替代信息。
- 键盘焦点在打开 Drawer/Modal 后进入内容，关闭后返回触发元素。

## 17. 页面验收清单

### 17.1 通用

- [ ] 已明确页面主模式，并使用对应模板。
- [ ] 页面或弹窗只有一个主色按钮。
- [ ] 桌面按钮使用 32 px，移动端按钮使用 40 px，触控热区不小于 44 px。
- [ ] 操作顺序符合“次要在左、主要在右”，按钮间距默认 8 px。
- [ ] 纯图标按钮有 Tooltip 和 `aria-label`。
- [ ] 异步操作有 Loading，并阻止重复提交。
- [ ] 无权限操作按规则隐藏，状态性不可用才禁用并说明原因。
- [ ] 正常、加载、空态、错误、刷新失败和无权限状态均已设计。
- [ ] 页面状态可通过 URL、浏览器历史或工作区上下文恢复。

### 17.2 查询、详情和编辑

- [ ] 查询区和列表区采用双 Card。
- [ ] 查询操作顺序为“重置、查询”。
- [ ] 表格行内直接动作不超过“查看、编辑、更多”。
- [ ] 普通详情使用 720 px Drawer；复杂详情使用独立页面。
- [ ] 画布检查器使用 480～560 px，不与普通详情混用。
- [ ] 复杂编辑采用独立页面，底部操作为“取消、保存草稿、提交审核”。
- [ ] 跨 Tab 表单能够定位首个错误 Tab。
- [ ] 未保存修改离开时有确认。

### 17.3 Tabs、看板和向导

- [ ] Tabs 只用于同一对象和任务下的并列内容，不表达流程步骤。
- [ ] 推荐 2～5 个 Tab，直接展示不超过 7 个，不嵌套 Tabs。
- [ ] Tab、筛选和分页可通过 URL 恢复。
- [ ] 看板 KPI 有明确口径、范围、更新时间和下钻行为。
- [ ] 图表局部失败不阻断整个看板。
- [ ] 向导每一步有完成判定、跳转条件和草稿恢复。
- [ ] 前置步骤变化会正确处理下游失效状态。

### 17.4 画布、导入和任务工作台

- [ ] 画布区与搜索区、工具栏和检查器职责清晰。
- [ ] 画布具备缩放、适配、定位、空态、错误和超大数据提示。
- [ ] 导入流程区分上传、预检、批次执行和结果处理。
- [ ] 错误行、警告行、重复行和有效行有明确语义。
- [ ] 后台任务支持进度、部分成功、失败明细和重试。
- [ ] 批量任务明确选择范围，并展示部分失败结果。
- [ ] 审核历史展示轮次、人员、时间、意见和耗时。

### 17.5 权限、审计、附件和安全

- [ ] 角色与数据范围同时展示，高风险授权有范围预览和二次确认。
- [ ] 权限变更原因必填，并有审计历史。
- [ ] 审计页面只读，敏感和技术字段按规则脱敏或折叠。
- [ ] 附件预览失败有下载兜底，删除前检查引用关系。
- [ ] 登录和找回密码不泄露账号是否存在。
- [ ] 安全表单支持正确自动填充并防止重复提交。
