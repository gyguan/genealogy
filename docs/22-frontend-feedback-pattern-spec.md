# 前端提示与反馈规范

> 适用范围：`frontend/genealogy-web` 全部业务页面与共享组件。
>
> 目标：减少提示类别、降低页面视觉噪声，并保证同一业务语义始终使用同一种反馈形式。

## 2. 统一后的五种形式

### 2.1 页面/区块状态：`PageFeedback`

适用于需要持续展示、用户必须感知的状态：

- 页面或区块加载失败
- 数据已过期、部分失败、结果被裁剪
- 权限限制
- 操作已提交但仍需后续处理

语义色：

- `info`：中性说明、权限限制、处理中的非阻断状态
- `success`：结果已成功完成，且有必要持续展示
- `warning`：结果可继续使用，但存在风险、不完整或待处理事项
- `error`：当前区域加载或处理失败，需要重试或修正

使用要求：

- 标题说明“发生了什么”，描述说明“原因或影响”，操作按钮说明“下一步”。
- 同一区块同时最多展示一个主提示；多个错误应合并摘要并提供查看详情。
- 不使用固定欢迎语、功能说明或重复业务规则占据页面。

### 2.2 字段辅助与校验：`Field hint/help`

仅适用于：

- 输入格式、范围、默认值
- 字段间依赖
- 当前字段即时校验错误

禁止用于：

- 页面加载错误
- 操作成功/失败
- 长篇业务说明
- 跨字段、跨区块的系统状态

### 2.3 短暂操作反馈：统一 Toast

适用于用户主动操作后的即时结果：

- 保存成功
- 删除成功
- 提交失败
- 下载失败

规则：

- 默认停留约 3 秒。
- 成功提示只写结果，不重复按钮名称。
- 错误提示优先给出可执行建议；需要持续处理的错误应转为 `PageFeedback`。
- 不同时混用 `notify`、Ant Design `message` 和自定义 Toast。

### 2.4 高风险确认：`ConfirmAction`

适用于删除、解除关系、覆盖、退出未保存页面等不可逆或高风险操作。

规则：

- 标题明确动作和对象，例如“删除附件《族谱卷一.pdf》？”
- 描述说明影响，不使用泛化文案“是否确认”。
- 危险操作的确认按钮使用危险态。
- 普通导航、查看详情、可撤销操作不弹确认框。

### 2.5 空状态：`EmptyState`

适用于查询结果为空、列表暂无记录或当前对象尚未维护数据。

空状态应区分：

- 初始空：提示如何开始，例如“选择宗族后查询”。
- 查询空：提示调整筛选条件。
- 权限空：使用 `PageFeedback`，不伪装成无数据。
- 加载失败：使用 `PageFeedback`，不伪装成无数据。

## 3. 两种受限辅助形式

### 3.1 Tooltip

仅用于图标按钮、缩写和不常见术语。页面上可直接看懂的标签不再增加 Tooltip。

### 3.2 FullPageFeedback / Result

仅用于整页无法继续：403、404、500、对象不存在或链接失效。局部请求失败不得占用整页 Result。

## 4. 文案规范

统一使用“状态 + 影响/下一步”的结构：

| 场景 | 推荐文案 |
|---|---|
| 加载失败 | `来源资料加载失败` + `请检查网络后重新加载。` |
| 数据陈旧 | `数据刷新失败，当前展示上次结果` + 错误摘要 |
| 权限不足 | `当前账号暂无附件上传权限` |
| 提交成功 | `引用变更已提交审核` |
| 查询为空 | `未找到符合条件的资料` + `请调整关键词或筛选条件。` |

避免：

- “温馨提示”“友情提示”“注意”作为无信息量前缀。
- 成功、信息、警告、错误语义与颜色不一致。
- 同一错误同时出现页面 Alert 和 Toast。
- 技术异常、堆栈、接口路径直接展示给用户。

## 5. 布局与密度

- 页面提示圆角统一为 8px，标题 14px，描述 13px。
- 页面级提示下方间距 16px，区块级提示下方间距 12px。
- 字段帮助文字 12px，不增加独立背景块。
- 空状态使用浅灰背景与虚线边框。
- Toast 固定右上角，最多同时显示 4 条。
- 移动端提示操作区允许换行，不使用固定宽度。

## 6. 迁移与治理

采用“统一外观、建立入口、存量递减”的策略：

1. 全局样式先统一现有 Alert、Form help、Empty、Result 和确认框外观。
2. 新增页面必须使用 `Feedback.tsx` 的标准组件。
3. 存量页面按高频顺序迁移：来源资料库、宗族文化、修谱工作台、成员权限、数据导入。
4. 审计基线记录各类历史机制数量；CI 只允许存量减少，不允许新增非标准形式。
5. 每完成一批迁移，同步下调基线。

已完成的首批迁移：

- `AsyncImportExecutionPanel.tsx`：页面状态、错误摘要、空状态和取消确认统一到标准组件。
- 删除该面板自定义 `import-panel-alert` 提示样式入口。
- 增加契约测试，防止该页面重新引入 `Alert`、`Empty`、`Popconfirm` 或自定义提示条。

## 7. 标准组件

```tsx
import {
  ConfirmAction,
  EmptyState,
  FullPageFeedback,
  InlineFeedback,
  PageFeedback
} from '@/shared/ui/Feedback';
```

- `PageFeedback`：页面/区块状态
- `InlineFeedback`：局部轻量状态
- `EmptyState`：空数据
- `FullPageFeedback`：整页不可继续
- `ConfirmAction`：高风险确认
