# Spec: 通用导入中心与页面内目标支派选择

## Objective

保留“导入管理”一级菜单，将当前人物导入页面升级为可扩展的通用导入中心；在导入页面内完成目标支派选择，用户无需先到其他页面设置工作区支派即可上传、预览和创建导入批次。

## Target Users

- 修谱主编
- 宗族管理员
- 支派管理员 / 支派编辑

## Scope

- `ImportPage` 作为通用导入中心，展示导入类型入口和统一导入任务区域。
- 人物导入作为当前已支持类型。
- 人物关系、字辈、来源资料导入以“规划中”状态展示，不提供不可用操作。
- 当前页面按宗族加载可操作支派，并提供目标支派下拉选择。
- 工作区支派仅作为首次进入的可选默认值，不再作为人物导入组件的前置依赖。
- 人物预览、创建批次全部使用当前页面明确选择的支派。
- 切换支派后清空已选择文件、预览和重复确认，防止误投。
- 通用导入任务区域与人物导入表单解耦，并跟随当前页面选择的支派过滤。

## Out of Scope

- 不新增关系、字辈或来源资料导入后端能力。
- 不修改导入 API、数据库结构或审核流程。
- 不拆分 `importType` 与 `fileFormat`，留待后续契约演进。
- 不新增宗族选择器，继续使用应用头部当前宗族上下文。

## Success Criteria

- 菜单名称继续为“导入管理”。
- 页面明确展示“人物导入”为已支持类型，其他类型标记为规划中。
- 未在其他页面选择支派时，用户仍可在导入页面加载并选择目标支派。
- 页面不再出现“请先在工作区选择目标支派”的提示。
- 文件上传、字段映射、预览和创建批次只依赖本页选择的目标支派。
- 支派切换后旧文件和预览不可继续提交。
- 创建批次后下方任务列表立即刷新，并按本页目标支派展示。
- 普通用户只看到支派名称，不直接操作支派 ID。

## Affected Files

- `frontend/genealogy-web/src/features/imports/ImportPage.tsx`
- `frontend/genealogy-web/src/features/imports/PersonImportWorkspace.tsx`
- `frontend/genealogy-web/src/features/imports/import-type-registry.ts`
- 导入模块定向验证和 Review 工件

## Security / Privacy

- 支派选择只影响前端请求上下文，后端继续执行最终支派权限校验。
- 文件中的支派 ID 仍不作为归属依据。
- 切换目标支派时清空临时文件和预览，避免跨支派误导入。
