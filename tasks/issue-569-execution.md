# Issue #569 执行看板

## 目标

审计追踪后端查询接口支持同名重复参数的多选 OR 查询，并与前端和 OpenAPI 保持一致。

## 任务

| 序号 | 任务 | 状态 | 说明 |
|---|---|---|---|
| 1 | 核对现有 Controller、Service、Repository 和 OpenAPI | ✅ | 已确认全部为单值参数 |
| 2 | 对象追踪支持对象类型、状态多选 | ✅ | 单值保留数据库原生分页，多值合并排序并全局分页 |
| 3 | 操作日志支持操作者、动作、对象、结果多选 | ✅ | 列表、统计和导出共用数据库 IN 过滤语义 |
| 4 | 风险事件支持多选 | ✅ | 数据库单次 IN 查询，保留原支派权限范围谓词 |
| 5 | 更新 OpenAPI 与测试 | ✅ | 新增 array + form/explode 契约和 Controller/Service 测试 |
| 6 | CI、Review 与合入 | 🔄 | Backend CI 与 API Contract 最终验证中 |

## 实现语义

- 同一筛选维度内为 OR，不同筛选维度之间为 AND。
- 空集合等价于未筛选；字符串值去空、去重、转小写并兼容逗号分隔值。
- 操作日志 `resultStatus` 映射到持久化的 `eventResult` 字段。
- 操作日志列表、统计和 CSV 导出使用同一组过滤条件。
- 风险事件多选在单次数据库查询中使用 `IN`，避免全选产生笛卡尔积查询。

## 约束核对

- 未改变登录、宗族成员、功能权限和支派数据范围判断。
- 未改变技术字段最小披露、导出权限和导出风险审计。
- 未改变隐私、分页上限和风险事件业务枚举。
