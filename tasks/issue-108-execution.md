# Issue #108 执行看板：失败行批量修正、重试与错误文件导出

- Issue：https://github.com/gyguan/genealogy/issues/108
- 分支：`agent/issue-108-bulk-failure-handling`
- 目标：提升大批量导入失败行的处理效率，在不覆盖原始数据、不绕过权限和审核的前提下，支持批量重试、批量排除、错误文件导出与修正文件回传。
- 首次开始：2026-07-14 17:56（北京时间）
- 最后更新：2026-07-14 17:56（北京时间）

## 实现范围

- 按勾选行或当前筛选条件批量重试失败行。
- 批量排除明确无效的失败行，强制填写排除原因并保留审计记录。
- 导出当前批次失败行，包含原始行号、稳定行键、原始数据、当前修正数据、错误码、错误说明和重试次数。
- 回传修正文件，以“批次内原始行号/稳定行键”匹配原行，不使用数据库行 ID。
- 使用行版本进行冲突检测，批量操作允许部分成功并返回逐行失败原因。
- 批量操作后重新计算批次统计与处理状态；排除数量进入批次和审核摘要。
- 前端提供筛选、跨页范围说明、勾选操作、导出、修正文件回传及部分失败结果展示。

## 非目标

- 不改变导入数据必须先进入草稿/批次并经过审核的业务红线。
- 不删除现有单行修正、单行重试接口和页面能力。
- 不开放跨批次批量操作。
- 不导出超出当前用户支派权限或现有隐私可见范围的数据。

## 影响模块

- 后端：`imports` 应用服务、DTO、Controller、Repository、行状态与审计。
- 数据库：失败行排除字段、稳定行键/查询索引（按现有模型实际缺口决定）。
- API：批量重试、批量排除、失败行导出、修正文件回传及批量结果契约。
- 前端：导入任务失败行列表、批量操作工具栏、文件导入导出和结果反馈。
- 测试：权限、状态门禁、版本冲突、部分失败、统计重算、导出字段和稳定键回传。

## 兼容与回滚

- 新接口增量提供，不删除或改变现有单行重试接口。
- 新字段采用可兼容默认值，历史失败行默认“未排除”。
- 原始 `raw_data` 永不覆盖；修正文件只更新当前修正数据并走版本冲突检测。
- 数据库迁移只前向新增；应用回滚时保留新增字段，后续使用更高版本补偿迁移。
- 批量操作设置明确上限并在数据库分页处理，避免无边界内存加载和长事务。

## 已知风险

- “按筛选条件全选”必须冻结筛选快照或明确作用范围，避免分页状态变化导致误操作。
- 批量重试可能触发重复查重或草稿创建，必须复用单行幂等规则。
- 导出与回传文件可能包含隐私字段，必须先执行后端支派权限与可见范围校验。
- 部分成功不能回滚已成功行，但必须返回未处理范围和逐行冲突原因。
- 排除行不能直接删除，审核摘要必须体现排除数量和原因。
