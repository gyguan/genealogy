## 最终实现范围

- 按勾选行或当前失败行范围批量重试，单次最多 500 行。
- 批量排除明确无效的失败行，强制填写排除原因并保留操作人、时间和原始数据。
- 导出当前批次失败行 XLSX，包含稳定行键、原始行号、原始数据、当前修正数据、错误码、错误说明、重试次数和版本。
- 回传修正文件，以 `row-<原始行号>` 匹配原行，不使用数据库行 ID。
- 使用行版本、任务锁和行锁进行冲突检测，批量操作允许部分成功并返回逐行失败原因。
- 批量操作后重新计算批次统计与处理状态；排除数量进入审核快照和摘要。
- 前端复用现有失败行列表，增加勾选、批量重试、批量排除、导出、回传和部分失败结果展示。
- 支持“全批次均排除”的审核生命周期，审核摘要明确展示草稿数与排除数。

## 非目标与边界

- 不改变导入数据必须先进入草稿/批次并经过审核的业务红线。
- 不删除现有单行修正、单行重试接口和页面能力。
- 不开放跨批次批量操作。
- 不导出超出当前用户支派权限的数据。
- 当前运行时已上线的人物、人物关系和来源资料导入支持完整批量重试；字辈导入当前仍标记为规划中，待其单行重试链路正式启用后接入统一批量框架。

## 五轴 Review 结论

### Correctness

- 批量操作逐行独立事务，版本冲突或业务失败只影响对应行。
- “全部失败行”在请求开始时冻结行号和版本快照；超过 500 行拒绝并要求分批处理。
- 修正文件使用稳定行键与原始行号双重校验，避免数据库 ID 泄漏或错误匹配。
- 全部行被排除时仍可提交审核并完成批准/驳回生命周期。

### Readability

- 外层编排、逐行执行器、Controller、DTO 和前端组件职责分离。
- 复用现有单行校验与草稿创建服务，不复制人物、关系和来源资料规则。

### Architecture

- 新接口和字段均为增量扩展，不改变现有单行修正契约。
- 批量编排框架可在新的导入类型单行重试链路启用后继续接入。

### Security

- 导出、回传、批量重试和排除均执行登录、宗族成员和支派写权限校验。
- 原始 `raw_data` 永不覆盖；排除保留原因、操作人和时间。
- 未知运行时异常不向前端暴露数据库或内部实现细节。

### Performance

- 单次批量处理上限 500 行。
- XLSX 导出采用数据库分页和 `SXSSFWorkbook` 流式写入。
- 任务与行采用数据库锁保证并发一致性，锁范围收敛到单行事务。

## 新增测试

- 勾选行批量处理部分成功与版本冲突。
- 当前全部失败行超过 500 条时拒绝处理。
- 排除证据、统计重算和原始数据不变。
- XLSX 导出稳定行键、原始数据、修正 JSON 和版本。
- 修正文件有效行与错误稳定键并行返回。
- 全批次均排除时的审核提交与摘要。
- 仅回传修正、不立即重试时的成功结果语义。

## 兼容与回滚

- 新接口增量提供，不删除或改变现有单行重试接口。
- 新字段采用可兼容空值，历史失败行默认“未排除”。
- 原始 `raw_data` 永不覆盖；修正文件只更新当前修正数据并走版本冲突检测。
- 数据库迁移只前向新增；应用回滚时保留新增字段，后续使用更高版本补偿迁移。
