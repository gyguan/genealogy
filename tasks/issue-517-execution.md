# Issue #517 执行看板

## 目标
移除建谱向导所有自动保存逻辑，仅保留用户点击“保存”按钮触发的显式草稿保存。

## 范围
- 移除输入事件触发的自动保存与草稿采集
- 移除自动保存定时器及卸载清理
- 移除步骤切换、审核提交和退出时的自动写入
- 移除向导容器输入与变更事件绑定
- 保留手动保存和已手动保存会话的恢复能力

## 非目标
- 不修改后端接口、数据库和建谱业务门禁
- 不改变步骤准入和业务校验逻辑

## 任务看板
- [x] 建立 Issue、任务分支与执行检查点

- [x] 移除自动保存逻辑并保留手动保存
- [x] 检查最终 diff

## 任务分类
- Issue 类型：前端缺陷修复
- 流程强度：轻量
- 契约强度：无 API 变更
- 验证强度：向导相关测试、TypeScript、生产构建
- 拆分结论：单一页面行为调整，不拆分

## 实现结果
- 删除 `autosaveTimer`、输入事件自动保存和卸载清理逻辑。
- 步骤切换不再采集或写入草稿。
- 提交审核后只标记当前状态未保存，不再自动写入。
- 退出向导不再提供“保存并退出”，未保存内容会明确提示丢失。
- `WizardShell` 不再绑定 `onInputCapture`、`onChangeCapture`。
- `persistSession` 仅由底部“保存”按钮调用。
