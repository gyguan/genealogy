- 目标：补齐禁用账号即时失效、旧会话重放审计、过期会话清理和多设备治理验收。

## 方案与影响

- 延续已批准的“服务端 Session + HttpOnly Cookie + CSRF”方案。
- 所有鉴权入口均校验账号状态，禁用账号不得继续使用尚未到期的 Session。
- 刷新已撤销 Session 时记录重放拒绝事件；随机无效 Token 不写入可识别敏感内容。
- 增加定时清理已过期及长期撤销会话，不改变活跃会话语义。

## 回滚

- 账号状态校验属于安全收紧，不提供生产关闭开关。
- 定时清理周期可配置；回滚应用时保留数据库字段与安全审计数据。
