## DEFINE

### 目标

将审计追踪页面重构为“审计追踪查询 Card + 查询结果 Card”双卡结构，三个 Tab 共用统一页面骨架，同时保留各自现有筛选、结果表头和详情能力。

### 范围

- 重组 `LogPage` 查询与结果页面结构。
- 将风险事件查询和结果逻辑并入统一页面组合，消除嵌套 Card。
- 增加统一多选筛选、更多筛选和响应式布局。
- 保持 URL、分页、导出、详情、权限、加载/空/错状态。
- 增加审计追踪聚焦模型测试和 Chromium E2E。

### 非目标

- 不修改 OpenAPI、后端、数据库、权限或审计业务语义。
- 不改变三个结果表格字段和详情内容。
- 不引入新依赖或第二套 UI 基础组件。

## 实现结果

1. 页面仅保留“审计追踪”和“查询结果”两个主 Card。
2. 查询 Card 内统一承载对象追踪、操作日志、风险事件三个 Tab。
3. 三个 Tab 默认均展示一行四项高频条件：
   - 对象追踪：对象类型、业务关键词、业务状态、最近变更时间；
   - 操作日志：时间范围、操作者、动作分类、对象类型；
   - 风险事件：时间范围、风险等级、事件类型、处置状态。
4. 更多筛选直接追加到同一 CSS Grid，不新增 Card、标题、Collapse 容器或分隔线。
5. 默认字段与展开字段统一使用 12px 行间距；按钮顺序固定为“更多筛选/收起、重置、查询”。
6. 所有枚举型下拉支持多选、搜索、全选和清空，URL 使用逗号保存多值状态；请求使用同名重复参数传递多值。
7. 结果 Card 标题统一为“查询结果”，按当前 Tab 切换原有结果表格，表头、分页、行点击、详情和对象追踪入口保持不变。
8. 操作日志的导出 CSV 移到结果 Card 右上角，仅在后端返回可导出技术字段时展示。
9. 390px 视口查询条件单列排列，更多筛选独占操作区第一行，重置与查询位于下一行。

## 失败修复记录

1. 首轮 Frontend CI 的 TypeScript 错误来自分组选项缺少顶层 `value`，已将多选组件选项类型调整为分组项可省略 `value`，类型和构建复跑通过。
2. 首轮 Chromium E2E 因 Ant Design 中文按钮可访问文本插入字间空格（`重 置 / 查 询`）导致文本数组断言失败；测试改为仅归一化空白，结构、顺序和间距断言未放宽，复跑通过。

1. OpenAPI 本次未变更；多选查询通过同名重复参数提交。前端模型测试已验证序列化，但未新增真实后端集成测试，后端参数绑定兼容性作为已知边界记录在 PR。
