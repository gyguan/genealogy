# Issue #803 执行记录

## 偏差结论

原实现通过 `index.html` 加载独立脚本，使用 `MutationObserver` 扫描 `.workbench-result-card` 并插入总览、质量检查和快捷入口。该方式无法与页面查询、分页、选择和任务状态共享 React 状态，且区块层级与确认原型不一致。

## 整改内容

- 新增 `EditingWorkspacePrototypePage`，由 React 正式渲染修谱工作台；
- 页面层级调整为：查询条件、工作台总览、当前任务与快捷入口、数据质量检查、修谱任务列表；
- 总览直接基于当前任务数据计算，不再扫描 DOM；
- 质量检查直接使用当前查询、已选草稿或整个修谱会话状态；
- 保留服务端提交审核门禁；
- 保留任务选择、批量核查、分页、导出、模板、详情 Drawer 和相关页面跳转；
- 删除 `workbench-enhancements.ts/css` 及其 HTML 启动入口；
- 增加原型层级、质量范围、响应式和 720px Drawer 聚焦测试。

## 非目标

- 不修改 #795/#796 后端质量规则；
- 不修改正式谱库数据；
- 不在审核中心恢复质量检查入口。
